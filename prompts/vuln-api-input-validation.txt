<critical>
**API INPUT VALIDATION ATTACK VECTORS:**
1. **Mass assignment** — Setting admin/role/internal fields via POST/PUT/PATCH that should be server-controlled
2. **Excessive data exposure** — API responses returning sensitive fields (passwords, tokens, PII) not needed by client
3. **Type confusion** — Sending arrays where strings expected, objects where primitives expected
4. **Parameter pollution** — Duplicate parameters with conflicting values, HPP attacks
5. **Content-type confusion** — Sending XML when JSON expected, multipart abuse
6. **File upload** — Unrestricted file types, path traversal in filenames, oversized uploads
7. **GraphQL-specific** — Deep nesting DoS, introspection exposure, batch query abuse
8. **Rate limiting** — Missing or bypassable rate limits on expensive operations
9. **Numeric limits** — Negative quantities, zero prices, integer overflow in calculations
</critical>
