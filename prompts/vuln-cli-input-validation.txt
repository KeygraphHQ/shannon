<critical>
**CLI INPUT VALIDATION ATTACK VECTORS:**
1. **Argument length/type abuse** — Oversized arguments, unexpected types, null bytes, Unicode edge cases
2. **Path traversal** — `../` sequences in file path arguments, symlink following, TOCTOU races
3. **Format string injection** — User input reaching printf-family functions or logging formatters
4. **Integer overflow/underflow** — Numeric arguments exceeding bounds, negative values where positive expected
5. **Glob/wildcard expansion** — Crafted filenames that expand dangerously, argument injection via filenames
6. **Config file parsing** — YAML/JSON/TOML deserialization vulnerabilities, type coercion issues
7. **Environment variable abuse** — Overlong values, injection via IFS, locale manipulation
8. **Resource exhaustion** — Inputs that trigger unbounded memory allocation, CPU loops, or disk writes
9. **Encoding attacks** — Mixed encoding exploits, BOM injection, null byte truncation
</critical>
