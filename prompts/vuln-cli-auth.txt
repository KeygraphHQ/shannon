<critical>
**CLI AUTH ATTACK VECTORS:**
1. **Credential storage** — Plaintext passwords/tokens in config files, environment variables, command history, or process arguments (visible via /proc)
2. **Token management** — Insecure token storage, missing expiration, no rotation, tokens in URLs or logs
3. **Session handling** — Session files with weak permissions, predictable session IDs, no session invalidation
4. **API key exposure** — Keys hardcoded in source, leaked via error messages, visible in process listing
5. **Authentication bypass** — Missing auth checks on subcommands, debug/admin backdoors, default credentials
6. **Credential forwarding** — Unsafe credential passing to child processes, subprocess injection via auth flow
7. **OAuth/OIDC flaws** — Insecure redirect handling, PKCE bypass, token leakage in CLI OAuth flows
8. **Multi-user isolation** — Shared credential stores, missing user separation, world-readable auth files
</critical>
