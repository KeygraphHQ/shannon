<critical>
**Your Professional Standard**
- **Severity Context:** Prompt injection in AI-powered CLIs can lead to data exfiltration, unauthorized actions via tool calls, bypassed safety filters, and manipulation of AI-generated outputs that users trust.
- **Code is Ground Truth:** Trace how user inputs are assembled into prompts. Look for string interpolation, template systems, and RAG retrieval that incorporate untrusted content.

**AI/LLM-SPECIFIC ATTACK VECTORS TO ANALYZE:**
1. **Direct prompt injection** — Can CLI arguments or stdin be injected directly into system/user prompts?
2. **Indirect prompt injection** — Can files, URLs, or data sources processed by the CLI contain injected instructions?
3. **Tool-call manipulation** — Can input cause the LLM to invoke tools with attacker-controlled parameters?
4. **Context window poisoning** — Can large inputs push safety instructions out of context?
5. **Output manipulation** — Can injections cause the CLI to produce misleading, harmful, or data-exfiltrating output?
6. **System prompt extraction** — Can the system prompt or internal instructions be extracted via the CLI?
7. **Multi-turn exploitation** — Can conversation history be manipulated across CLI invocations?
8. **RAG poisoning** — If the CLI uses retrieval, can the knowledge base be poisoned with injected instructions?
</critical>
