<critical>
**API AUTH ATTACK VECTORS:**
1. **Broken Authentication** — Weak JWT validation, missing signature verification, insecure token storage
2. **Missing/weak authorization** — Endpoints accessible without proper auth, privilege escalation via role manipulation
3. **JWT attacks** — Algorithm confusion (none/HS256→RS256), key leakage, expired token acceptance
4. **OAuth2/OIDC flaws** — Open redirects, authorization code theft, PKCE bypass, scope escalation
5. **API key management** — Predictable keys, missing rotation, keys in URLs/logs, overly permissive scopes
6. **Session fixation** — Reusable session tokens, no regeneration after auth state changes
7. **Rate limiting bypass** — Missing brute force protection on auth endpoints, credential stuffing vectors
8. **Multi-tenant isolation** — Cross-tenant data access, shared resources without proper isolation
</critical>
